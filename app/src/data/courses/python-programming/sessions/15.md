## Phase 1 — Learn (30 min)

**Why this matters:** A CLI tool is a real, usable piece of software. By the end of this session, you'll have built something you can run from your terminal, share with others, and extend. This session pulls together everything from the course: file I/O, classes, exception handling, modules, and now argparse for a proper command-line interface.

- **argparse** — Python's built-in library for building CLI tools with proper argument parsing, auto-generated `--help`, and type validation.
- **ArgumentParser** — create one with `parser = argparse.ArgumentParser(description="What this tool does")`.
- **add_argument()** — add each flag: `parser.add_argument("--file", help="Path to input file", required=True)`. Use `type=int` to auto-convert to integer.
- **parse_args()** — `args = parser.parse_args()`. Then access values as `args.file`, `args.output`, etc.
- **Subparsers** — for tools with multiple commands (like `git add`, `git commit`), use `parser.add_subparsers()`.
- **--help for free** — argparse auto-generates a `--help` output from your `description` and `help=` strings. No extra work.

> **Key insight:** Run `python tool.py --help` before showing your tool to anyone. If the help text clearly explains what the tool does and what arguments it takes, you've built it well. The help text is your documentation.

## Phase 2 — Do (75 min)

Build a **Todo List CLI** that saves to a JSON file:
- `python todo.py add "Buy milk"` — adds a task
- `python todo.py list` — shows all tasks with numbers
- `python todo.py done 2` — marks task #2 as complete
- `python todo.py clear` — removes all completed tasks
- Data persists in `todos.json` between runs (use `json.load` / `json.dump`)
- Use a `TodoItem` class with `task` and `done` attributes
- Wrap file operations in try/except for graceful failure

**Alternative options** if the todo list doesn't appeal to you:
- Currency converter using the Open-Meteo-like free currency API
- A script that renames files in a folder matching a pattern

## Review

Run `python todo.py --help`. Does it explain itself? Does each subcommand have a clear description? If not, add `help=` strings to your `add_argument()` calls until it does.
