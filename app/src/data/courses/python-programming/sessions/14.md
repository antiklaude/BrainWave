## Phase 1 — Learn (40 min)

**Why this matters:** Now that you've used libraries like `requests` and `pandas`, you've seen inheritance in action without knowing it — every time you called a method from a class you imported. Inheritance is how libraries are built. It lets you extend existing behaviour without duplicating it. The BankAccount from Session 07 is about to evolve.

- **Inheritance** — a child class inherits all methods and attributes from a parent class. `class SavingsAccount(BankAccount):` means `SavingsAccount` gets everything `BankAccount` has, plus whatever it adds.
- **super().__init__()** — inside the child's `__init__`, call `super().__init__()` to run the parent's setup code. Otherwise the parent's attributes won't exist on the child.
- **Method overriding** — if the child defines a method with the same name as the parent, the child's version runs. The parent's is still accessible via `super().method_name()`.
- **Why inheritance** — don't repeat shared logic. If `BankAccount` and `SavingsAccount` both need `deposit()` and `withdraw()`, put them once in `BankAccount` and inherit.
- **__str__** — `def __str__(self): return f"Account({self.owner})"` controls what `print(obj)` shows. Without it, you get `<__main__.BankAccount object at 0x...>`.
- **__repr__** — a developer-facing representation used in the REPL and logging. Define both when building classes meant to be used by others.

> **Key insight:** Inheritance is about "is-a" relationships. A `SavingsAccount` IS A `BankAccount`. A `Dog` IS AN `Animal`. If the relationship doesn't fit "is-a", use composition (one object holds another) instead of inheritance.

## Phase 2 — Do (40 min)

- Extend `BankAccount` from Session 07 with a `SavingsAccount` subclass that adds:
  - `interest_rate` attribute (e.g. 0.05 for 5%)
  - `apply_interest()` method that increases the balance by `balance * interest_rate`
  - Override `__str__` to show `"SavingsAccount(owner, balance, rate%)"`
- Create instances of both `BankAccount` and `SavingsAccount`, call `apply_interest()` on the savings account, and verify inheritance by calling `deposit()` on the savings account (it should work without being redefined)

## Review
