## Phase 1 — Learn (40 min)

**Why this matters:** No real Python project uses only built-in functions. You'll always need external libraries — for HTTP requests, data analysis, web scraping, and more. Modules and pip are how Python's ecosystem of 400,000+ packages becomes available to you. Virtual environments are how you keep projects from interfering with each other.

- **import** — `import math` makes the `math` module available. Access its functions with `math.sqrt(16)`.
- **from X import Y** — `from math import sqrt` lets you call `sqrt(16)` without the prefix. Useful but can create naming conflicts.
- **Standard library modules** — Python ships with `math`, `datetime`, `random`, `os`, `json`, and hundreds more. No installation needed.
- **Creating your own module** — any `.py` file is a module. `utils.py` with a function can be imported with `import utils`.
- **pip install** — `pip install requests` downloads and installs the `requests` library from PyPI. `pip list` shows all installed packages.
- **requirements.txt** — `pip freeze > requirements.txt` saves all installed packages. Anyone cloning your project runs `pip install -r requirements.txt` to reproduce your environment.
- **venv** — `python -m venv venv` creates a virtual environment in a folder called `venv`. Activate it: `source venv/bin/activate` (Mac/Linux) or `venv\Scripts\activate` (Windows). Your prompt changes to show `(venv)`.

> **Key insight:** Always create a `venv` before `pip install`-ing anything for a project. Your future self (and any teammates) will thank you. A requirements.txt without a venv is a car with no engine.

## Phase 2 — Do (40 min)

- Create a new project folder with a `venv` inside. Activate it.
- Install the `requests` library: `pip install requests`
- Run `pip freeze > requirements.txt` and open the file — note what's in it
- Add `venv/` to a `.gitignore` file (create it if it doesn't exist)
- Create a `utils.py` file with one function, then import and use it from a `main.py` file in the same folder
