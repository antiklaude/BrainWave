## Phase 1 — Learn (40 min)

**Why this matters:** You will hit errors every single session. Developers who can read an error traceback and understand it in 10 seconds move 10× faster than those who don't. This session gives you two things: string manipulation tools you'll use constantly, and the ability to diagnose your own bugs.

- **String indexing** — each character has a position starting at 0. `s = "Python"` → `s[0]` is `"P"`, `s[-1]` is `"n"` (negative counts from the end).
- **Slicing** — extract a substring: `s[1:4]` returns characters at index 1, 2, 3 (not 4). Think of it as `[start:stop]` where stop is excluded.
- **Common string methods** — all return a new string, the original is unchanged:
  - `.upper()` / `.lower()` — change case
  - `.strip()` — remove leading and trailing whitespace
  - `.split(",")` — split into a list on a separator
  - `.replace("old", "new")` — substitute all occurrences
- **Reading a traceback** — Python prints error messages bottom-up. The last line is the most important: it gives the error type and a plain-English description. The line above it tells you exactly which file and line number caused it.
- **NameError** — `name 'x' is not defined` — you used a variable that doesn't exist yet. Check for typos.
- **TypeError** — `unsupported operand type(s)` — you tried to do something to a type that doesn't support it (like `"hello" + 5`). Fix: `"hello" + str(5)`.
- **IndentationError** — Python uses indentation to mean "this code belongs to this block." Wrong indentation breaks the program.

> **Key insight:** When you see an error, read the last line first, not the first line. The last line tells you what went wrong. The middle lines tell you the call stack that led there — usually irrelevant for beginners.
