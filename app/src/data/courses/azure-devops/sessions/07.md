## Phase 1 — Learn (40 min)

**Why this matters:** Every team eventually builds shared internal libraries — utility code, design system components, internal SDKs. Publishing these to public package registries (PyPI, npm) would expose proprietary code. Azure Artifacts is a private package registry that integrates directly with your pipelines and your team's existing tooling.

- **Azure Artifacts feeds** — a feed is a private package registry. Your team publishes packages to it; other projects consume them. One feed can host multiple package types (pip, npm, NuGet, Maven).
- **Creating a feed** — Artifacts → Feeds → Create Feed. Choose visibility (organisation-scoped = only your org).
- **Upstream sources** — configure your feed to proxy PyPI (or npm). Now `pip install requests` from your feed first checks if requests is cached in your feed; if not, it fetches from PyPI and caches it. This provides a consistent, auditable package supply chain.
- **Publishing a package** — use `twine upload` (Python) or `npm publish` (Node) pointed at your feed URL with authentication.
- **Consuming in a pipeline** — authenticate using a service connection and install from the feed:
  ```yaml
  - task: TwineAuthenticate@1
    inputs:
      artifactsFeed: 'my-feed'
  - script: pip install mypackage --index-url https://pkgs.dev.azure.com/...
  ```
- **Package versioning** — use semantic versioning (major.minor.patch). Bump the version before publishing. Azure Artifacts stores all versions; consumers can pin to a specific version.

> **Key insight:** The real value of Azure Artifacts is not hosting your packages — it's the upstream source caching. Even if you publish nothing, configuring Artifacts as a proxy for PyPI means your builds never fail because PyPI has an outage. Cached packages are always available.

## Phase 2 — Do (40 min)

- Create an Artifacts feed called `internal-packages`
- Configure PyPI as an upstream source
- Create a minimal Python package (`setup.py` or `pyproject.toml` with a simple utility function)
- Publish the package to your feed using `twine`
- In a new pipeline step, install your package from the feed and verify it's available
