## Phase 1 — Learn (40 min)

**Why this matters:** Pull Requests are how professional teams review code before it lands in the main branch. A PR is not just a "merge request" — it's a structured conversation about the code, a chance to catch bugs, share knowledge, and maintain standards. Branch policies are the enforcement mechanism that makes PR reviews mandatory.

- **Creating a PR** — in Azure Repos, after pushing a branch, click "Create a pull request". Choose the source branch (your feature branch) and target branch (main). Add a clear title and description.
- **PR description** — describe what changed and why. Include a link to the work item. "Added login form" is bad. "Adds email/password login form with basic validation — closes #42" is good.
- **Adding reviewers** — tag specific people who should review the code. They receive a notification.
- **Code comments** — reviewers can comment on specific lines. Authors respond and address feedback.
- **Approving** — reviewers click "Approve" when they're satisfied. With policies enabled, the merge button unlocks after the required approvals.
- **Completing a PR** — click "Complete" to merge. Options: merge commit (default, preserves history), squash merge (one clean commit), rebase.
- **Branch policies** — in Azure Repos → Branches → your branch → Branch policies:
  - **Require a minimum number of reviewers** — prevent self-merge
  - **Check for linked work items** — every PR must reference a Boards work item
  - **Check for comment resolution** — all PR comments must be resolved before merge
  - **Require a successful build** — the CI pipeline must pass

> **Key insight:** Branch policies are not bureaucracy — they're quality guarantees. A team of 1 doesn't need them. A team of 5 absolutely does. Set them up once and they protect your main branch automatically forever.

## Phase 2 — Do (40 min)

- Configure branch policies on `main`:
  - Enable "Require a minimum number of reviewers: 1"
  - Enable "Check for linked work items" (optional but good practice)
- Create a PR from your `feature/hello-world` branch to `main` with a clear title and description
- Try to merge directly to main by pushing — verify that the policy blocks it
- Approve and complete the PR — observe the merge in the Git history
