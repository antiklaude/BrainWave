## Phase 1 — Learn (45 min)

**Why this matters:** A deployed application that nobody is watching is an accident waiting to happen. Monitoring tells you when things go wrong before users do. Secret management tells you how to give your application the credentials it needs without ever writing a password in code. Together, they're the difference between a demo and a production system.

- **Application Insights** — Azure's application performance monitoring (APM) tool. Connect it to an App Service and it automatically collects: request counts, response times, failure rates, exceptions (with stack traces), and dependency call durations.
- **Live Metrics** — shows real-time request rates, server response time, and failed requests as they happen. Your first stop when something goes wrong.
- **Failures tab** — shows all exceptions and failed requests. Click any exception to see the full stack trace, which request triggered it, and how many times it occurred.
- **Variable groups** — in Pipelines → Library → Variable Groups. Create a group, add variables. Reference in YAML:
  ```yaml
  variables:
    - group: my-variable-group
  steps:
    - script: echo "API base URL: $(API_BASE_URL)"
  ```
- **Azure Key Vault integration** — link a Key Vault to a variable group. Secrets in the vault appear as variables in the pipeline — but their values are never visible in logs (masked with `***`).
- **Secret masking** — any variable marked as secret in a variable group is automatically masked in pipeline logs. Test this: `echo $(MY_SECRET)` in a pipeline step will print `***`.

> **Key insight:** Secrets must never appear in pipeline logs. If a secret appears in a log, it's compromised — the log may be stored in Azure DevOps, audit systems, and SIEM tools accessible to many people. Use Key Vault + masked variables. This is the only correct approach.

## Phase 2 — Do (45 min)

- In Azure Portal, create an Application Insights resource and link it to your App Service
- Make several requests to your app and observe them in Application Insights Live Metrics
- Deliberately cause an error (call a route that doesn't exist) — find the exception in the Failures tab
- Create a Variable Group with one plain-text variable (e.g., APP_ENV=production) and one linked Key Vault secret
- Reference the variable group in your pipeline — verify the secret is masked in the logs
