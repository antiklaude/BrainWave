## Phase 1 — Learn (50 min)

**Why this matters:** Colour is not material. A red plastic, a red painted metal, and a red ceramic all look completely different because they interact with light differently. PBR (Physically Based Rendering) materials simulate how light physically bounces off surfaces, producing results that look realistic under any lighting. This session teaches the Principled BSDF — one node that replaces dozens of separate material calculations.

- **Shader Editor** — accessible from the top dropdown (or press Shift+F3). All materials are built as node graphs here. Nodes connect via sockets — coloured dots on the left (inputs) and right (outputs) of each node.
- **Principled BSDF** — the master material node. Simulates almost any real-world surface using these key inputs:
  - *Base Color* — the surface colour or albedo texture (what colour the surface is)
  - *Roughness* — 0 = mirror, 1 = matte chalk. Controls spread of reflections.
  - *Metallic* — 0 = non-metal (plastic, wood), 1 = metal (iron, gold). Changes how highlights behave.
  - *Normal* — a normal map input for adding surface detail without extra geometry
  - *Alpha* — transparency (requires material blend mode set to Alpha Blend in Material Properties)
- **Image Texture node** — loads a texture image (Add → Texture → Image Texture). Connect it to any input on the Principled BSDF. The node reads UV coordinates from the mesh to know which pixel maps where.
- **Normal Map node** — normal map images encode surface direction in RGB. They must be decoded by a Normal Map node (Add → Vector → Normal Map) before connecting to the Principled BSDF's Normal input. Always set the Image Texture's Color Space to Non-Color for roughness and normal maps.
- **Poly Haven** — polyhaven.com is a free PBR texture library. All textures are CC0 (public domain). Download texture sets as ZIP (includes colour, roughness, normal, AO maps). This is the standard free resource for Blender texture work.
- **Connecting a full PBR set**:
  1. Add Image Texture → load colour map → connect to Base Color
  2. Add Image Texture → load roughness map → set Color Space to Non-Color → connect to Roughness
  3. Add Image Texture → load normal map → set Color Space to Non-Color → add Normal Map node → connect to Normal

> **Key insight:** The three inputs that control 90% of how a material looks are Base Color, Roughness, and Metallic. Master these three before touching anything else. A wooden plank: colour texture + roughness 0.6 + metallic 0 = convincing wood. A steel beam: grey colour + roughness 0.2 + metallic 1 = convincing metal.

## Phase 2 — Do (50 min)

Apply a full PBR wood material to the crate from Session 05 (which you UV-unwrapped in Session 07):
1. Go to polyhaven.com → Textures → filter by "Wood" → download a plank or crate-style wood texture as a ZIP (1K or 2K resolution)
2. Extract the ZIP — find the colour (diffuse), roughness, and normal map images
3. In Blender, open the Shader Editor with the crate selected
4. Add an Image Texture node → load the colour map → connect to Base Color
5. Add a second Image Texture node → load the roughness map → set Color Space to Non-Color → connect to Roughness
6. Add a third Image Texture node → load the normal map → set Color Space to Non-Color → add a Normal Map node in between → connect Normal Map output to Normal input
7. Switch to Material Preview (Z) to see the result — check that the wood grain aligns with the UV unwrap
8. Optionally: add a second material slot for the metal bolts, set Metallic = 1, Roughness = 0.3
