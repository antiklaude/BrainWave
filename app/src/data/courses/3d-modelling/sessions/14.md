## Phase 1 — Learn (50 min)

**Why this matters:** Rigging is the process of building a skeleton inside a character so it can be posed and animated. Without a rig, animating a character requires manually moving every vertex. With a rig, you move bones and the mesh follows. This session covers the foundational concepts — armatures, bone hierarchy, parenting, weight painting — that apply to all rigging work regardless of complexity.

- **Armature** — the skeleton object in Blender. Add one with Shift+A → Armature. An armature contains bones; bones are the individual segments that deform the mesh.
- **Bones — Edit Mode** — press Tab to enter Edit Mode on the armature. Bones have a *Head* (ball at start), a *Body*, and a *Tail* (pointed end). Extrude new bones from the tail (E). Each bone can be the parent of another, creating a hierarchy. The hierarchy matters: rotating a parent bone moves all its children.
- **Bone naming** — name bones clearly (e.g., "upper_arm.L", "upper_arm.R"). Blender's mirror tools use `.L`/`.R` suffixes for symmetry. In Edit Mode, select all bones on one side and use Armature → Symmetrize to mirror the rig.
- **Pose Mode (Ctrl+Tab)** — the mode where you animate. Rotating a bone in Pose Mode deforms the mesh. Resetting poses: Alt+G (location), Alt+R (rotation), Alt+S (scale).
- **Parenting the mesh to the armature** — the key step:
  1. Select the mesh
  2. Shift+click the armature (armature becomes active)
  3. Ctrl+P → With Automatic Weights
  Blender calculates which bones influence which vertices based on proximity.
- **Weight Painting** — how to fix bad automatic weights:
  - Select the mesh in Object Mode, then go to Weight Paint Mode (from the Mode dropdown, or Ctrl+Tab with the mesh selected while the armature is in Pose Mode)
  - Select a bone in the armature
  - Red areas = strongly influenced, Blue = not influenced
  - Paint with Add/Subtract/Smooth brushes to correct the influence
- **Inverse Kinematics (IK)** — a constraint that lets you move the endpoint of a chain (e.g., the hand) and have all the parent bones (forearm, upper arm) follow automatically. Add via Bone Constraints → Inverse Kinematics. Set Chain Length to define how many bones are in the IK chain.

> **Key insight:** Automatic weights are a starting point, not a finished rig. Always check the deformation by posing extreme positions (full arm raise, deep squat) and fix weight painting where the mesh pinches or stretches incorrectly. This is where rigging time really goes.

## Phase 2 — Do (50 min)

Rig a simple low-poly character arm:
1. Model or use a basic arm shape — a cylinder with 3 segments (upper arm, forearm, hand)
2. Shift+A → Armature → add 3 connected bones inside the arm geometry
   - Bone 1: upper arm (shoulder to elbow)
   - Bone 2: forearm (elbow to wrist)
   - Bone 3: hand (wrist to fingertips)
3. Name bones: "upper_arm", "forearm", "hand"
4. Parent mesh to armature: select mesh → Shift+click armature → Ctrl+P → With Automatic Weights
5. Switch armature to Pose Mode, rotate the forearm bone — mesh should follow
6. Check Weight Paint: does the upper arm area deform correctly? Is there unwanted influence at the shoulder? Paint corrections if needed.
7. Optional: add an IK constraint to the hand bone — set target to an Empty object (Shift+A → Empty → Plain Axes), move the Empty and watch the whole arm follow
