## Phase 1 ‚Äî Learn (45 min)

**Why this matters:** The DOM (Document Object Model) is the browser's live representation of the HTML on the page. JavaScript can read and change any part of it at any time ‚Äî updating text, toggling styles, creating new elements. This is how every interactive feature on the web works.

- **document.querySelector(selector)** ‚Äî finds the first matching element using any CSS selector. Returns `null` if not found.
- **document.querySelectorAll(selector)** ‚Äî returns a NodeList of all matches. Loop with `forEach()`.
- **element.textContent** ‚Äî read or set the visible text of an element. Always prefer this for displaying content ‚Äî it treats the value as plain text, never as markup.
- **element.classList** ‚Äî `.add('class')`, `.remove('class')`, `.toggle('class')` ‚Äî great for toggling visible states without writing inline style changes.
- **element.style** ‚Äî `element.style.backgroundColor = "red"`. Use for dynamic, programmatic styles; use `classList` for predefined states.
- **createElement + appendChild** ‚Äî create elements with `document.createElement('div')`, set their `textContent`, and add to the page with `parent.appendChild(newEl)`. This is the safe way to add dynamic content.
- **addEventListener(event, callback)** ‚Äî `button.addEventListener('click', () => { })`. Events: `click`, `input`, `change`, `submit`, `keydown`, `mouseover`.
- **event.target** ‚Äî inside an event handler, `event.target` is the element that triggered the event.
- **event.preventDefault()** ‚Äî prevents the browser's default behaviour (e.g., stop a form from submitting and refreshing the page).

> **Key insight:** When displaying user-provided content, always use `textContent` or `createElement` + `appendChild`. This ensures the content is treated as plain text and cannot affect page behaviour ‚Äî a critical habit for building secure websites.
