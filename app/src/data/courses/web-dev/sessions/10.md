## Phase 1 — Plan (20 min)

**Why this matters:** You've learned all the pieces. This session connects them into a real, deployable page. Planning before coding is the skill that separates people who produce finished products from people who get stuck in the middle.

Before writing code, sketch on paper or a whiteboard:
- What sections does the page have? (e.g., nav, hero, features, testimonials, footer)
- What colours and fonts will you use? Pick them now — don't decide as you code.
- What's the call to action? Every landing page has one primary thing it wants the visitor to do.
- How does it look on mobile? Sketch both desktop and mobile layout.

## Phase 2 — Build (85 min)

Build a complete, responsive landing page with these sections:
1. **Sticky navigation bar** — logo on the left, links on the right (Flexbox), `position: sticky; top: 0`
2. **Hero section** — large headline, one-line description, CTA button. Vertically centred using Flexbox or Grid.
3. **Features section** — 3 cards in a CSS Grid (`repeat(auto-fill, minmax(260px, 1fr))`), each with an icon (use emoji or text), title, and description
4. **Testimonials** — 2 quotes in a Flexbox row with a name and title below each
5. **Footer** — copyright, links, social icons

Requirements:
- The viewport meta tag must be present
- Must look good on mobile (stack everything with media queries at 768px)
- Use CSS variables for your colour palette
- At least one hover transition on a button or card

## Review

View your finished page on your phone (or press F12 → toggle device toolbar → select a phone). Does anything break or overflow? Fix it before moving on.
