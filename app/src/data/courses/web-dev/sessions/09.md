## Phase 1 — Learn (45 min)

**Why this matters:** Every modern web app fetches data from a server — loading user profiles, searching products, getting live weather. JavaScript does this asynchronously: it sends a request, continues doing other things, and handles the response when it arrives. Async/await makes this code look almost like regular synchronous code.

- **Asynchronous** — JavaScript doesn't wait for one thing to finish before starting the next. When you fetch data, the browser sends the request and moves on. The callback runs when the response arrives.
- **fetch(url)** — the browser's built-in HTTP request function. Returns a Promise.
- **Promise** — an object representing an eventual value. Either resolves (success) or rejects (failure).
- **async function** — `async function getData() {}` makes a function return a Promise and allows `await` inside it.
- **await** — `const response = await fetch(url)` — pauses the current function until the Promise resolves. The syntax looks synchronous but works asynchronously.
- **response.json()** — also async, also needs `await`: `const data = await response.json()`.
- **try / catch** — wraps `await` calls to handle errors: network failures, bad JSON, non-200 responses.
- **Checking response status** — `if (!response.ok) throw new Error(response.statusText)` before calling `.json()`.

> **Key insight:** `response.json()` doesn't give you the data — it gives you a Promise that will give you the data. Always `await` it. Forgetting this `await` is one of the most common bugs when learning fetch.

## Phase 2 — Do (45 min)

- Use the JSONPlaceholder API (free, no key, no rate limits) at `https://jsonplaceholder.typicode.com/posts`:
  - Fetch the list of posts
  - Dynamically create a `<div>` for each post with its title and body, and append them to the page
  - Add a loading spinner (show while fetching, hide when done)
  - Handle the error case: show a user-friendly "Failed to load posts" message if fetch fails
- Wrap everything in try/catch
