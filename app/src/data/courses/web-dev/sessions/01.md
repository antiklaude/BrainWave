## Phase 1 — Learn (40 min)

**Why this matters:** HTML is the skeleton — CSS is the skin, JavaScript is the muscle. Nothing works without a solid skeleton. And a skeleton built with the right bones (semantic elements) is understood by screen readers, search engines, and developers who maintain your code after you.

- **<!DOCTYPE html>** — the first line of every HTML file. Tells browsers to use modern HTML5 standards. Without it, old rendering quirks may apply.
- **Document structure** — `<html>` wraps everything. `<head>` contains invisible metadata (title, charset, CSS links). `<body>` contains the visible content.
- **Meta charset** — `<meta charset="UTF-8">` in the head. Without it, special characters like é, ü, or 中文 may display as garbage.
- **Semantic elements** — HTML5 introduced tags that describe the meaning of content, not just its position:
  - `<header>` — the top of a page or section
  - `<nav>` — navigation links
  - `<main>` — the primary content (use only once per page)
  - `<section>` — a thematic grouping with a heading
  - `<article>` — self-contained content (blog post, card)
  - `<footer>` — the bottom of a page or section
- **Headings** — `<h1>` to `<h6>`. One `<h1>` per page. Never skip heading levels — screen readers use them to navigate.
- **Paragraphs and links** — `<p>` for text blocks, `<a href="url">` for links. Use `target="_blank"` to open in a new tab (add `rel="noopener"` for security).
- **Lists** — `<ul>` (unordered, bullets), `<ol>` (ordered, numbers), `<li>` for items.

> **Key insight:** Using `<div>` for everything works, but tells browsers and screen readers nothing. Using `<nav>` tells every accessibility tool exactly what that block is. Write HTML for the machine that reads it, not just the human who sees it.

## Phase 2 — Do (40 min)

- Build a personal bio page (`index.html`) from scratch with:
  - Correct DOCTYPE and document structure
  - A `<header>` with your name as an `<h1>`
  - A `<main>` with a `<section>` for "About Me" and a `<section>` for "My Skills"
  - An unordered list of 5 skills
  - A `<footer>` with a link to your email (`mailto:`)
- Open it in a browser — does it render correctly?
