## Phase 1 — Learn (40 min)

**Why this matters:** Forms are how users send data to your application — search queries, login credentials, contact messages, file uploads. Getting forms right (including accessibility) is one of the most impactful skills in web development. Tables are for tabular data, not layout.

- **Form structure** — `<form>` wraps all inputs. `action` is where data goes on submit. `method` is `GET` (data in URL) or `POST` (data in body).
- **Input types** — `type="text"` (generic text), `type="email"` (validates email format), `type="number"` (numeric keyboard), `type="password"` (masked), `type="checkbox"` (true/false), `type="radio"` (one of many).
- **Label and for** — `<label for="name">Name</label><input id="name">` — the `for` value matches the input's `id`. Clicking the label focuses the input. Required for accessibility.
- **Textarea** — `<textarea rows="4" name="message">` for multi-line text. Has opening and closing tags unlike `<input>`.
- **Select and option** — `<select>` creates a dropdown. Each choice is an `<option value="value">Label</option>`.
- **Button types** — `type="submit"` submits the form, `type="button"` does nothing by default (useful for JavaScript), `type="reset"` clears the form.
- **Images** — `<img src="path.jpg" alt="Description" width="300">`. `alt` is mandatory for accessibility. `width`/`height` prevent layout shifts.
- **Tables** — for data that has rows and columns. `<table>` → `<thead>`/`<tbody>` → `<tr>` (row) → `<th>` (header cell) or `<td>` (data cell).

> **Key insight:** Never use tables for layout — that was a 1990s hack. Tables are for data like spreadsheets or schedules. Use CSS Grid or Flexbox for page layout.
