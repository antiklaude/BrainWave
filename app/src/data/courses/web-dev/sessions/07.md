## Phase 1 — Learn (45 min)

**Why this matters:** HTML and CSS build the page. JavaScript makes it respond. Without JavaScript, clicking a button does nothing, a form field doesn't validate, and the page is completely static. JavaScript is a full general-purpose programming language — not just a styling helper.

- **let / const** — `const` for values you won't reassign (use this by default), `let` for values that will change. Never use `var` in modern JavaScript.
- **Types** — string, number, boolean, null (deliberately empty), undefined (not yet set), object, array.
- **Template literals** — backtick strings with `${expression}` for interpolation: `` `Hello, ${name}!` ``
- **if / else** — same concept as other languages. `===` (strict equality, checks type + value), `!==` for not equal.
- **for loop** — `for (let i = 0; i < 5; i++) {}`. Or `for (const item of array) {}` for iterating arrays.
- **Arrow functions** — `const add = (a, b) => a + b`. Short syntax for functions. The `=>` replaces `function` keyword.
- **Arrays** — `const fruits = ["apple", "banana"]`. Key methods:
  - `.push()` — add to end
  - `.pop()` — remove from end
  - `.map()` — transform every element, returns new array
  - `.filter()` — keep elements that pass a test, returns new array
  - `.find()` — return first element that passes a test
- **Objects** — `const person = { name: "Alice", age: 30 }`. Access with `person.name` or `person["name"]`.

> **Key insight:** `===` checks both value AND type. `5 === "5"` is `false`. `==` does type coercion and causes bugs. Always use `===` in JavaScript.

## Phase 2 — Do (45 min)

- Build a **tip calculator** in a `.js` file:
  - Input: bill amount and tip percentage (as variables or `prompt()`)
  - Calculate: tip amount and total
  - Output with template literals: `"Bill: $50, Tip (15%): $7.50, Total: $57.50"`
- Use `.map()` to convert an array of prices from USD to GBP (multiply by 0.79)
- Use `.filter()` to keep only prices above 10
